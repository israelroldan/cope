Product Overview

CompoundWay takes a vague desire to improve — "I want to get more fit," "I want to be more creative," "I want to be a better partner" — and transforms it into a structured, adaptive, forgiving system of daily practice missions.

Unlike existing habit trackers that assume users already know what to do, CompoundWay solves the upstream problem: it figures out what the user should do, designs complementary practice tracks, generates scaffolded daily missions, guides execution, captures proof and reflection, and sustains engagement through streaks with built-in forgiveness.

MVP Scope: Two of Three Layers

v1 Habit Tracks

AI-generated systems of complementary practice tracks based on user input. Cadence management, load balancing, progressive mission generation.

v1 Missions

Individual practice sessions powered by the Guided Achievement Engine's five-phase pipeline: refinement, scaffolding, execution, reflection, engagement.

v2 Identity Layer

Earned identity narrative, "you're becoming" milestones. v1 collects the data (completion history, 3×3 matrix); v2 surfaces the story.

Target User

Adults who want to improve in some area of their life but struggle to convert that intention into consistent action. They may have tried other habit apps and churned because those apps assumed they already knew what to do and just needed reminders.

Onboarding Flow

The onboarding is the product's first impression and primary differentiator. It must feel warm, conversational, and radically different from typical "set up your habits" workflows.

Design reference: Her (film) energy — not a chat UI, but card-and-bubble interactions that feel like a conversation.
1

Welcome Screen

Warm greeting, no sign-up wall. Single soft prompt: "What would you like to spend more time on?" Free text input + "Inspire me" button.

2

Refinement Loop

Verb + object detection determines specificity. If vague: progressive narrowing via tappable option bubbles. Max 3 rounds (server-enforced). Context preserved via selectionPath.

3

Track Presentation

System generates 2–3 complementary tracks as cards. Shows track name, frequency, time per session. Recommends a starting subset. User picks all, some, or just one.

4

First Mission (Immediate)

"Want to try your first one right now?" Full scaffolding: tinyStart, materials, doneLooksLike. Timer, completion, proof, confetti celebration.

5

Dashboard Reveal

After first mission, show the main dashboard with tracks laid out. Prove the value before showing the system.

6

Account Creation

Prompted after first mission (not before). Apple Sign-In + email. Anonymous session data migrated to account.

Track Management

Track Detail View

Name, description, cadence. Completion history calendar. 3×3 assessment distribution. Current challenge level. Pause / resume / archive controls.

Streaks & Weekly Pass

The streak tracks consecutive days with at least one completed mission. The weekly pass is the critical retention mechanism — it lets users restore a broken streak once per week, solving the "I missed one day and now my 30-day streak is gone" churn cliff.

🔥 Streak Display

Calendar view with consecutive days forming connected "pills." Current streak count and best streak (all-time high watermark) prominently displayed.

🎟️ Weekly Pass

One per ISO week, auto-resets each Monday. Offered when streak breaks and there's a streak worth saving. Restores previous streak value. Visual availability indicator.

💔 Streak Break

If user misses a day: show last streak value, offer weekly pass if available. If declined or unavailable: streak resets to 1 on next completion. No guilt messaging.

🧠

AI Refinement Engine

The engine converts vague user input into specific, actionable habit tracks with missions. It operates at two levels:

Two-Level Refinement

Track-Level

Takes broad input ("I want to get fit") and produces a set of 2–3 complementary habit tracks with cadences and session lengths.

Mission-Level

Takes a track's context and generates a specific mission card for today's session, with full scaffolding fields.

Discriminated Response Types

Every AI call returns exactly one of three shapes: TrackPlan (set of complementary tracks), Refinement (narrowing question with 2–4 tappable options), or Card (fully scaffolded mission). The branching rule is verb + object detection.

Server-Side Guardrails

Round 3+ forces output regardless of model response. Fallback track plan / card built from raw input if model fails. Input sanitized against injection. Rate limits per user.

Special Behaviors

🔄 Passive → Active

"Watch TikTok" → "Film a 30-second tutorial." The system redirects passive consumption toward active skill-building creation.

🎯 Natural Filter

Unscaffoldable goals ("win the lottery") get gently redirected: "That's a great goal — what skills would help you get there?"

🔗

Resource Search & Validation

Provides real, working helper links — never hallucinated URLs.

1

Search Agent

Grounded web search tool returns 4–5 candidate resources. Agent explicitly prohibited from fabricating URLs.

2

Server-Side Validation

Extract URLs, resolve redirects, validate liveness (YouTube via oembed, others via HEAD with 3s timeout). SSRF protection: reject localhost, private IPs, non-HTTPS.

3

Trim & Deliver

Trim to 3 verified resources. Delivered async — card appears first, resources follow. If ALL validations fail, return originals (graceful degradation).

Streak System — Logic

Core Streak Calculation (Server-Side)

On mission completion: If yesterday has a completion → streak = old + 1. Otherwise → save last streak, reset to 1. Best streak = max(best, current). All dates in ISO format (YYYY-MM-DD). Computed server-side to avoid timezone drift.

Weekly Pass Logic

ConditionDetail
EligibilityweeklyPassAvailable === true AND lastStreakBeforeReset > 0
EffectStreak restored to lastStreakBeforeReset + 1 (restored + today)
ConsumptionweeklyPassAvailable set to false
ResetAuto-resets to true each new ISO week (Monday)

Visualization

Calendar view with consecutive days forming connected visual "pills." Colored dots per completed mission on each date (using track color). Current and best streak prominently displayed. Weekly pass availability indicator visible.

🎼

Track Orchestration

📅 Cadence Management

Distributes missions across the week. Avoids scheduling same-category tracks on consecutive days. Respects user-defined rest days. Even distribution across available days.

⚖️ Load Balancing

Max 2 missions per day across all active tracks. Rotates which tracks fire each day. User can always manually trigger additional missions.

📈 Progressive Generation

When a track fires: pull track context → call mission AI → generate Card. Difficulty informed by completion count (v1) and 3×3 matrix (v2).

Critical design rule: Completion is never reverted on server failure. The user's sense of accomplishment is protected even if the network is unreliable.

Design Direction

Warm, not clinical. Rounded corners, soft shadows, approachable typography. Card-based. Missions, tracks, completions — all cards. Minimal chrome. Let content breathe. Color per track. System-assigned from curated palette. Conversational tone. "Nice work" not "Task completed."

Risks & Mitigations

AI generates low-quality tracks or missions

Core value prop fails, users churn. Mitigation: Extensive prompt engineering, server-side validation, fallback cards, beta testing with rapid prompt iteration.

Resource URLs are broken or irrelevant

Erodes trust. Mitigation: Validation pipeline with liveness checks, SSRF protection, graceful degradation.

Onboarding feels too long

Users drop off before first mission. Mitigation: Hard stop at 3 refinement rounds, immediate mission after track selection, measure and optimize funnel.

Users create too many tracks → overwhelm

Burnout, churn. Mitigation: Load balancing (max 2/day), system recommends starting subset, progressive track addition.

Streak break causes churn despite weekly pass

Lose users after bad week. Mitigation: Weekly pass, no guilt notifications, soft re-engagement messaging.

AI API latency affects UX

Mission generation feels slow. Mitigation: Cache missions ahead of schedule, async resource loading, optimistic UI.

Offline edge cases break streak calculation

Frustration, data inconsistency. Mitigation: Server-side streak logic with ISO dates, client sync on reconnect, completion never reverted.
