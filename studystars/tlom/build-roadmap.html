Key Insights

Two parallel critical paths

Content creation (education expert) and platform build (technical person) run simultaneously. They don't block each other until pilot launch, when both must be ready.

Content before code

Module 1 content (#15 → #16 → #17) can start on Day 1 with zero technical infrastructure. The education expert only needs Google Docs and Claude.

Manual before automated

Artifact processing (#13), content pipeline (#12), and WhatsApp (#8) are all manual during MVP. Automate only after learning what "good" looks like from real cohorts.

Full PRDs are post-pilot

Don't build production systems (#1-#9) until the pilot validates demand. Sprint 4-6 only execute if pilot succeeds. This keeps risk low and learning high.

The #4 Content Layer is the keystone

Content Layer PRD (#4) unlocks three downstream prompts (#5, #6, #9). It's the single most important production PRD — prioritize it in Sprint 4.

Insight Discovery comes last

#7 (Insight Discovery) needs real facilitator and parent data. Don't build it until you've run 3+ cohorts. The compounding knowledge architecture only works with real signal.
